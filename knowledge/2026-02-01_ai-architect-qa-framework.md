## 要点

AI変革アーキテクト（コンサルタント）としての営業・提案活動を支援するQ&A資料の設計フレームワーク。CFO・CTO・新規事業責任者の3ペルソナに対し、各20問の想定質問と回答テンプレートを体系化した。回答は「結論→背景→解決アプローチ→期待成果→次のアクション」の5段構成で統一。

## 詳細

### 3つの顧客ペルソナ

| ペルソナ | 役職 | コアペイン |
|---|---|---|
| A: 田中誠（56歳） | 東証プライム製造業CFO | AI投資5億円でROI不明、PoC7件全て本番化失敗 |
| B: 佐藤健太（42歳） | SaaS CTO | ML/AIは専門外、DS採用決まらず、PoC→本番化が毎回失敗 |
| C: 山田麻衣（38歳） | 金融機関 新規事業部長 | 技術的実現可能性が不明、社内稟議が遅い、孤独な戦い |

### 質問カテゴリ設計（ペルソナ別各20問）

- **ペルソナA**: ROI・投資対効果(5) / 人材・組織(4) / PoC・推進(5) / 戦略・意思決定(4) / リスク・契約(2)
- **ペルソナB**: 技術選定(5) / 本番化・運用(5) / チーム・採用(4) / アーキテクチャ(4) / 最新動向(2)
- **ペルソナC**: 事業企画・実現可能性(5) / 社内調整・稟議(5) / ベンダー・パートナー(4) / PoC・事業化(4) / 自己成長(2)

### 回答テンプレート（1問1ページ）

```
■ 結論（2-3行）: 端的に結論 + ペインへの共感
■ 背景・分析（3-5行）: 問題の原因、業界傾向
■ 解決アプローチ（10-15行）: 3-5アクションステップ + 事例1つ
■ 期待される成果（2-3行）: 定量的な目安
■ 次のアクション（1-2行）: 今すぐできる一歩
```

### 回答のトーン

- 自信と謙虚さのバランス / 専門用語には平易な説明を添える
- 各回答に実績事例を1つ含める（Amazon累計7億円、ROAS 30%改善等）
- 「まず明日からできること」を必ず明示

- 提案書・プレゼン資料の回答構成パターンとして
